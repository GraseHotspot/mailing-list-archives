> Hi All,
>

We finally figured out where we were going wrong.

On machine B, we disbaled the automatic metric and set its value for the 
primary interface to 1. Also, the service named was running on port 53 
which was needed by dnsmasq. So we freed that port for dnsmasq and it 
worked!!

Is there some place where we can document it, so that if anybody else faces 
these issues, he can find the solution easily.

Thanks,
